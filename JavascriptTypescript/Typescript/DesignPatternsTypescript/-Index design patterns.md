# Design Patterns

Um padrão de projeto é uma solução já pronta para problemas bem cotidianos. São soluções documentadas para resolver um problema ou evitar esses problemas comuns.

Design Patterns são muito bons para resolver mais rapidamente e mais eficientemente os problemas, já que são soluções muito bem testadas e muito bem*classes em estruturas* maiores) e **Comportamental** (*C produzidas. Dessa forma, podemos dividir os padrões de projeto em três categorias: 
Temos o **Padrão Criacional** (criação escalável de *objetos*), **Estrutural** (Trabalhando com omunicação* eficiente e gerenciamento de *responsabilidades*)

Não força padrões de projeto. Não adianta, só usa o remédio se precisar. Outra coisa, não tem time de padrão de projeto, não existe preferência, usa quando for usável e presta atenção se os outros desenvolvedores sabem sobre os padrões, não adianta usar os granfinos se isso piora a comunicação.
